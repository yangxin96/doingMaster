sys_role 角色表
sys_role_resources  角色权限中间表
sys_resources 权限表

说明：三张主标表，2张中间表，通过ID在中间表进行关联，主表的ID都保存在中间表里面，可以通过左链接或者又连接进行关联。
inner join 是内连接；比如a表某ID和b表某ID在中间表是相等的


SELECT sys_resources.user_name FROM sys_user LEFT JOIN sys_user_role ON sys_user.id=sys_user_role.user_id
LEFT JOIN sys_role ON sys_user_role.role_id=sys_role.id
LEFT JOIN sys_role_resources ON sys_role.id=sys_role_resources.role_id
LEFT JOIN sys_resources on sys_role_resources.resources_id=sys_resources.id
LEFT sys_user.id=8

替换表名写法

SELECT sys_resources.user_name FROM sys_user a LEFT JOIN sys_user_role b ON a.id=b.user_id
LEFT JOIN sys_role c ON b.role_id=c.id
LEFT JOIN sys_role_resources d ON c.id=d.role_id
LEFT JOIN sys_resources e on sd.resources_id=e.id
LEFT a.id=8

# 单点登录

三种方式

1、session广播机制实现（session复制）（不推荐）
1
2、使用cookie+redis实现
 	每次发送请求，带着cookie值进行发送
 	(1) 在项目任何模块登录，登录之后把数据放到两个地方
 			1、redis,在key生成唯一随机值（ip、用户ID等等），在value：用户数据
 			2、cookie，把redis里面生成key值放到cookie 里面
 	（2）访问项目其他模块，发送请求带着cookie进行发送，获取cookie值，拿着cookie做事情：
  		1、把cookie获取值，到redis里面查询，根据key查询，如果查询到数据就登录

3、使用token实现（按照一点规则生成字符串，字符串包含用户信息）
	令牌
	(1)、用户在项目某个模块登录之后按照规则生成字符串，用户信息在字符串里面，把字符串返回。
	 		1、字符串通过cookie返回
			2、字符串通过地址返回
	(2)、用户再访问其他模块，每次访问，地址栏都会带着登录生成的字符串，在访问的模块里面获取地址栏字符串，如果可以获取到字符串里面的用户信息就是登录

# jwt

jwt生成的字符串包含三部分

第一部分：jwt头信息

第二部分：有效载荷，包含主体信息，（用户信息）

第三步的：前面哈希，防伪标志

package com.example.navi.config;
import io.jsonwebtoken.Claims;
import io.jsonwebtoken.Jws;
import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.SignatureAlgorithm;
import org.springframework.http.server.reactive.ServerHttpRequest;
import org.springframework.util.StringUtils;

import javax.servlet.http.HttpServletRequest;
import java.util.Date;

/**
 * @author helen
 * @since 2021/11/29
 */
public class JwtUtils {

    //常量
    public static final long EXPIRE = 1000 * 60 * 60 * 24; //token过期时间
    public static final String APP_SECRET = "ukc8BDbRigUDaY6pZFfWus2jZWLPHO"; //秘钥

    //生成token字符串的方法
    public static String getJwtToken(String id, String nickname){

      //构建字符串
        String JwtToken = Jwts.builder()
                .setHeaderParam("typ", "JWT")
                .setHeaderParam("alg", "HS256")
									//
                .setSubject("nave-user")
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + EXPIRE))

                .claim("id", id)  //设置token主体部分 ，存储用户信息
                .claim("nickname", nickname)
                //可添加多个nickname（昵称）
//                .claim("nickname1",nickname)
//                .claim("nickname2",nickname)
//                .claim("nickname3",nickname)
                .signWith(SignatureAlgorithm.HS256, APP_SECRET)
                .compact();

        return JwtToken;
    }

    /**
     * 判断token是否存在与有效
     * @param jwtToken
     * @return
     */
    public static boolean checkToken(String jwtToken) {
        if(StringUtils.isEmpty(jwtToken)) return false;
        try {
            Jwts.parser().setSigningKey(APP_SECRET).parseClaimsJws(jwtToken);
        } catch (Exception e) {
            e.printStackTrace();
            return false;
        }
        return true;
    }

    /**
     * 判断token是否存在与有效
     * @param request
     * @return
     */
    public static boolean checkToken(HttpServletRequest request) {
        try {
            String jwtToken = request.getHeader("token");
            if(StringUtils.isEmpty(jwtToken)) return false;
            Jwts.parser().setSigningKey(APP_SECRET).parseClaimsJws(jwtToken);
        } catch (Exception e) {
            e.printStackTrace();
            return false;
        }
        return true;
    }

    /**
     * 根据token字符串获取会员id
     * @param request
     * @return
     */
    public static String getMemberIdByJwtToken(HttpServletRequest request) {
        String jwtToken = request.getHeader("token");
        if(StringUtils.isEmpty(jwtToken)) return "";
        Jws<Claims> claimsJws = Jwts.parser().setSigningKey(APP_SECRET).parseClaimsJws(jwtToken);
        Claims claims = claimsJws.getBody();
        return (String)claims.get("id");
    }
}
