# mysql容器持久化

准备工作：

docker pull mysql
1

创建宿主机持久化文件夹

mkdir -p /vcar/local/mysql/data
1

挂载主机目录时出现的一个问题，确认语句正确情况下但是容器始终运行不起来

原因是CentOS7中的安全模块selinux把权限禁掉了

解决博客 (opens new window)

镜像启动后，我们启动mysql服务，发现mysql无法启动。查找官方文档，因SElinux服务开启，需要在宿主机执行如下命令：

chcon -Rt svirt_sandbox_file_t /var/local/mysql
1

或者关闭SElinux也可以

selinux服务对root用户权限进行控制，很多企业中:selinux服务默认关闭

# 1.1、临时关闭服务

这三个命令只是临时生效，系统重启后恢复

  1. 临时关闭SELinux
#setenforce 0
1

2、临时打开SELinux

#setenforce 1
1

3、查看SELinux的状态

#getenforce
1

# 1.2、永久关闭

vi /etc/selinux/config

SELINUX=disabled
1

PS: 如果想让selinux配置文件生效,重启系统

SELinux 服务有三种配置模式，具体如下。

➢ enforcing：强制启用安全策略模式，将拦截服务的不合法请求。

➢ permissive：遇到服务越权访问时，只发出警告而不强制拦截。

➢ disabled：对于越权的行为不警告也不拦截。

Selinux，默认情况下，就已经将已知的一些网络服务默认要访问的路径添加到规则库，并打上相应的context。

# ps axZ

上述配置完成后，仍然无法启动mysql，结合mysql日志查看如果是文件权限的问题，在宿主机上给予共享文件夹对应的权限：

给文件夹赋予读写权限

chmod 777 -R /var/local/mysql/data
1

再启动

docker run -itd --name mysqltest -v /var/local/mysql/data:/var/lib/mysql/data -e MYSQL_ROOT_PASSWORD=root --privileged=true --net=host mysql

--privileged=true  赋予容器拥有root权限，对宿主机有读写权限
--net=host       宿主机与容器共用一个端口号

如果上面命令不行：
docker run -itd --name mysqltest -v /var/local/mysql/data:/var/lib/mysql/data -e MYSQL_ROOT_PASSWORD=root mysql

# 数据库自增重新开始

自增ID重新开始会一般会清空表数据。

四种方法：

  • ALTER TABLE 重置法：适合表格刚刚被打乱，乱序的位置之后的数据量不多的情况
  • TRUNCATE TABLE 重置法：适合表格中数据不重要，可清空重来。
  • 保留数据重置 - 直接删除 ID 法：数据量较小时。(不推荐使用)
  • 保留数据重置 - 两步替换 ID 法：前一种方法的保守方法，数据更安全。

1、使用 ALTER TABLE 重置自增 ID：

ALTER TABLE table_name AUTO_INCREMENT = value;
1
  • table_name 为当前要重置自增 ID 的表名。
  • value 必须是大于当前表中最大 ID 数，比如当前表目前ID最大数为 2，我们可以设置成 3。

我们使用以下命令将 table_name 自增 ID 重置为 3

ALTER TABLE table_name AUTO_INCREMENT = 3;
1

2、使用 TRUNCATE TABLE 重置自增 ID

TRUNCATE TABLE 原理是删除掉表中全部数据并释放空间，达到重置 ID 的效果，但它与 DROP TABLE （直接删除表）不同，它不会删掉表本身，也不会删掉表的定义，仅清空表数据。

TRUNCATE TABLE table_name;
1

3、直接删除 ID 法（不推荐使用）

此方法的原理是直接把 ID 列删掉，然后重新新建 ID 列，从而达到保留数据，重置 ID 的效果

这种方法对于数据量较大的表有一定风险。此方法在删除 ID 后，重新加入新 ID 时，有可能顺序会被打乱，保险起见，我们可以使用另一种在保护数据的前提下，重置 ID 的方法。

我们先删除 ID 列：

ALTER  TABLE table_name DROP id;
1

然后再插入新 ID 列：

ALTER  TABLE  table_name ADD id mediumint(6) PRIMARY KEY NOT NULL AUTO_INCREMENT FIRST;
1

最后用 SELECT 看一下表的变化：

SELECT * FROM table_name;
1

4、ID替换法

为了保证删除 ID 列后，再加入新 ID 列这种方法不会打乱原有数据的顺序，我们可以使用更保守一些的替换法，达到重置 ID 的效果。

首先，取消 table_name 表中 ID 列的自增和主键的定义：

ALTER TABLE table_name MODIFY COLUMN id int NOT NULL FIRST ,DROP PRIMARY KEY;
1

在表里新增 id_renew 列，设置为自增（AUTO_INCREMENT）主键（PRIMARY KEY），我们将使用这个列来替换 ID 列。

ALTER TABLE table_name ADD COLUMN id_renew int NULL AUTO_INCREMENT FIRST ,ADD PRIMARY KEY (id_renew);
1

使用 SELECT 查看当前表情况，可以看到新增的 id_renew 这一列的顺序已经是连续的了。

接着，我们可以删除掉原来乱序的 ID 列

ALTER TABLE table_name DROP COLUMN id;
1

最后，我们将 id_renew ，重命名，更为 id

ALTER TABLE table_name CHANGE COLUMN id_renew id int(7) NOT NULL AUTO_INCREMENT FIRST;
1

在保护数据以及数据顺序不被打乱的前提下，我们重置了 ID 列的数字顺序。

# 问题

mysql报错：1044 -Access denied for user ‘root‘@‘%‘ to database
1

出现这个错误的原因是因为创建数据库之后没有进行授权

解决办法

1.进入mysql 容器

docker exec -it mysql bash
1

2.输入账号密码

mysql -uroot -p
1

权限

grant all on xxxx.* to 'root'@'%' identified by 'password' with grant option;

其中,xxxx代表所操作数据库名称，password为访问数据库账户的密码

或者输入下面命令
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%';
